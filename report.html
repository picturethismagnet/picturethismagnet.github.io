Summary — The Big Picture

Hi Lily — this report covers everything that went into building the PictureThisMagnet website from scratch. The site is a fully custom, hand-coded static website that connects directly to your Shopify store for product data and checkout.

No Shopify theme, no website builders — every page was designed and coded specifically for PictureThisMagnet. The result is a fast, accessible, mobile-friendly site that you own and control through GitHub Pages.

What you're getting

❌ Before
  • Default Shopify storefront
  • Limited design control
  • Generic look shared with other stores
  • No custom pages (FAQ, events, wholesale)
  • No accessibility testing
✓ After
  • 12 custom-designed HTML pages
  • Unique brand identity with curated colors & fonts
  • Dedicated pages for events, wholesale, FAQ
  • WCAG AA accessible (tested with axe-core)
  • Mobile-optimized with 44px touch targets
  • Automated product sync every 6 hours
  • Live Shopify cart integration

Automatic Product Sync

A Python script fetches your product catalog from Shopify every 6 hours and saves it locally. This means your site always shows up-to-date products, prices, and availability — even if Shopify's API is temporarily slow.

Sync architecture

Shopify Store
picturethismagnet.myshopify.com
→
sync-products.py
(GitHub Actions)
→
_data/products.json
(committed to repo)
→
GitHub Pages
auto-deploys

Runs every 6 hours at 00:00, 06:00, 12:00, 18:00 UTC — or manually anytime

Client-side fallback

The browser also fetches live data directly from Shopify when the page loads. If Shopify is unreachable, it falls back to the locally-synced _data/products.json. This dual approach ensures products always display.

JavaScript// Primary: fetch live from Shopify const res = await fetch(`https://${SHOPIFY_DOMAIN}/products.json`); // Fallback: use synced static data const fallback = await fetch('/_data/products.json');

Accessibility Compliance (WCAG AA)

Every page passes automated accessibility testing against WCAG 2.0 and 2.1 at the A and AA conformance levels. This means the site works well for people using screen readers, keyboard navigation, and assistive technologies.

What was tested

  • Color contrast — all text meets 4.5:1 minimum ratio (body) and 3:1 (large text)
  • Keyboard navigation — all interactive elements reachable via Tab key
  • ARIA labels — buttons, nav toggle, cart drawer, and dialogs properly labeled
  • Semantic HTML — proper heading hierarchy, landmark regions, list structures
  • Image alt text — all images have descriptive alt attributes
  • Form labels — every input has an associated label, even if visually hidden
✅ Result
All 11 tested pages pass axe-core with zero violations. Tests run automatically via Playwright to catch regressions.

Mobile Compatibility
